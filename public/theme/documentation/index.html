A) Twitter Bootstrap & Less CSS - top

Photon UI Admin Panel Theme utilizes Twitter Bootstrap as a foundation. Twitter Bootstrap is built with Less CSS. In order to preserve original Twitter Bootstrap files, instead of changing the originals, needed changes were done by overriding Twitter Bootstrap less files with Photon less files. This means that you will find original Bootstrap less files in css/less_bootstrap folder and Photon less files in css/less_photon folder. Both need to be included in order for proper UI display. Any CSS files supplied by third party JS plugins were also converted to less format and placed in css/plugins folder.

B) PHP Sources & Configuration Files - top

Please note that aside from static HTML version Photon UI Admin Panel Theme is shipped with PHP sources as well. If using PHP sources please set-up a local WAMP, LAMP or MAMP environment. Theme uses configuration files to minimize code repetition and to allow easier testing. Configuration files have the .ini extension, and follow the standard php configuration files formatting (more info at http://php.net/manual/en/function.parse-ini-file.php ).

/library/config.ini

config.ini is used to specify page titles, appearance of "Add New" button, parent menu items (used to render breadcrumbs), and to configure items in User Navigation Menu.

/library/menu.ini

menu.ini is used to configure items in Main Navigation Menu.

folder: /library/jstree/

Files inside library/jstree_menu/ folder are used to configure Main Menu. If Main Menu includes jsTree tree plugin, both .php and .ini files are required in order to set-up the menu for the specific theme page. File names must be exactly the same as their parent theme page names in order to automatically load.

C) HTML Sources & Localhost Development environment - top

Please note that due to the usage of various third party javascript plugins that rely on AJAX technology, cross-domain issues are raised while trying to load the Photon UI Theme without a localhost development environment. Please install WAMP http://www.wampserver.com/en/, MAMP http://www.mamp.info/en/index.html or similar package, copy all the source files in htdocs root, and than run the file in browser so that the path looks somewhat like this: http://localhost/photonui/html_sources/index.html .

Furhter on, please note that due to the fact that IE 8 & 9 have a limitation on the number of CSS selectors that can be placed in a single CSS file (more info about the issue: http://blesscss.com/), you can not use .less files to code/develop in those browsers. Recommended workflow is:

  1. Make nececery adjustments in modern browser (Latest IE, Chrome or Firefox)
  2. Compile less files into CSS (more info at: http://lesscss.org/)
  3. Use Bless CSS http://blesscss.com/ to split the files into smaller files, or do it manually

Another important notice is that HTML source files, as opposed to PHP source files, use pre-compiled CSS that has already been split in parts, but was not minified so that users who don't want to use .less files can edit these to tweak the theme. These files are:
/css/css/photon.css
/css/css/photon-pt2.css
/css/css/photon-photon-responsive.css

D) Navigation Menus Structure - top

Photon UI Admin Panel Theme features two navigation menus:

Main Navigation Menu

Main Navigation Menu is a two level menu that slides in and out on mouseover. The menu is included from the file library/common/main_menu.php.

User Navigation Menu

User Navigation Menu is included from the file library/common/user_menu.php.

E) Dashboard - top

Dashboard page features Quicklaunch Bar and Widget boxes.

Quicklaunch Bar

Quicklaunch Bar's width is CSS contolled to adapt to the user's screen size. Items can be reordered via jQuery UI sortable plugin. Modal dialog to add new items can be launched by clicking the plus icon.

Widgets

Widgets have fixed width of 302px. Depending on a screen size and number of included widgets, there can be one or more widgets in a row. Widget settings can be accessed by clicking on a gear icon that is visible upon hovering over a widget area. Each widget can also be used on the inside pages in the Main Menu. Dashboard widgets can be reordered via jQuery UI sortable plugin. For the sake of simplicity each widget is stored as a separate php file in library/widgets/ folder and can be easily included via php include function (as seen in dashboard.php file). Blank widget is also available, and may be used as a template to create your own widgets, or a simple placeholder in a Dashboard.

F) Inner Pages - top

All inner template pages share common structural elements:

  • Main Navigation Menu (described in section above)
  • User Navigation Menu (described in section above)
  • Main Menu
  • Breadcrumbs Strip
  • Content Area

Main Menu

Main menu displays a logo, module name (with the optional "Add New" button to the right), and an universal search field. In most cases main menu has a jsTree tree menu below the search field. Tree serves as an infinite-level menu that can be used to link deep inside the module structure. Tree menu is skinned to render in a touch-friendly size for tablet and mobile devices. Below the tree, any number of Widgets can be placed. Widget width will match the upper elements width (universal search, Title and 'Add New' button). Entire menu scrolls independently with the help of the Custom Content Scroller jQuery plugin. Main menu can be closed and the jQuery Cookie plugin is used to remember it's state.

Breadcrumbs Strip

Breadcrumbs strip uses xbreadcrumbs jQuery Plugin. Breadcrumbs are styled differently for desktop and mobile screen sizes.

Content Area

Since Photon UI Admin Panel Theme is built upon Twitter Bootstrap, the Content Area uses fluid container div elements with 'container-fluid' class applied to organize content in separate units. Bootstrap's 12 grid system is also utilized along with the control group sections. Each row of elements must be contained within a div element with applied classes 'control-group row-fluid'.
